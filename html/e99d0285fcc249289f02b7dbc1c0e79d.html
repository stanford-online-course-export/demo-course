If you are creating an A/B test between components in a single unit, you'll want to find the vertical file that corresponds to the unit.  This vertical file will contain two component (these could be html, problem or video) url names that will correspond to the components in that unit.  Please click here to view detailed instructions on how to create this kind of A/B test.

Creating an A/B test between components within a unit:

  1. Go to Studio and create your section, subsection, and unit.
  2. Add two components to this unit.

    Note: They can be the same kind of component or different. There can also be multiple components within a unit.

    Here is an image of what you should have at this point in Courseware:

  3. Export your course by clicking on Tools > Export and then clicking on the blue button to export your course's content.

  4. Unzip the exported .tar.gz file you receive.
  5. Open an editing program, such as TextWrangler or SublimeText.
  6. Open the folder that contains all of your course's files.
  7. In Studio, go to the unit that you want to create the A/B test in.  Every unit will have a Unit Location module in Studio.  Note the Unit Identifier string for this unit.  Set this information aside for the moment.

  8. Now go to the live version of your course and go to the corresponding unit.
  9. For the components in question, click on "Staff Debug Info".  Copy the location URL.  The location URL will start with "i4x://" and may look something like this:

    i4x://StanfordOnline/OpenEdX/problem/821c9feed6594c0b8fea90e97a7b33bd

  10. Go back to the editing program and find the string in the vertical folder of your course file that corresponds to the Unit Identifier string you noted in step 7. Open this file.

  11. Insert a <split_test> tag after the <vertical> tag.  Close this with a </split_test> tag before the closing </vertical> tag.

    Here is how your vertical file might look before editing it:

    <vertical display_name="split3">
      <problem url_name="821c9feed6594c0b8fea90e97a7b33bd"/>
      <problem url_name="50938f7b9ca64d0d9956f8ff46f781f4"/>
    </vertical>

    Here is how the above vertical file looks after I've inserted a <split_test> tag:

    <vertical display_name="split3">
        <split_test user_partition_id="0" group_id_to_child='{"0": "i4x://StanfordOnline/OpenEdX/problem/821c9feed6594c0b8fea90e97a7b33bd", "2": "i4x://StanfordOnline/OpenEdX/problem/50938f7b9ca64d0d9956f8ff46f781f4"}'>
              <problem url_name="821c9feed6594c0b8fea90e97a7b33bd"/>
              <problem url_name="50938f7b9ca64d0d9956f8ff46f781f4"/>
        </split_test>
    </vertical>
  12. In layman's speak, the above is basically saying, "We have a user partition with the name (or ID) of "0" specified elsewhere.  This user partition has two children, "0" and "2".  For the child "0", show them this component at this location ID.  For the child "2", show them this component that exists at this other location ID."
  13. Save your change to the vertical file after you make the above edits.
  14. Zip your file back up as a .tar.gz file.
  15. Import this file back into your course by going to Tools > Import.
  16. When you are done, your unit in Studio will no longer be editable and will look something like this:

  17. The end result in Courseware will look something like this:

  18. Students will not see the drop-down menu. Rather, students who end up (note this assignment is randomly assigned and cannot be specified) in one group will see the component specified for that group, and students in the other group will only see the component that has been specified for that group.
  19. If you export and edit your course file again at a later time, you might notice that the vertical file you edited and imported now looks different.
