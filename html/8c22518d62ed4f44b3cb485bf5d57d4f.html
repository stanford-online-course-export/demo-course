If you are creating an A/B test between subsections in a section, you'll want to find the chapter file that corresponds to the section.  This chapter will contain the sequential url names that will correspond to the subsections in that section.  Click to expand detailed instructions for creating this kind of A/B test.

Creating an A/B test between subsections within a section:

  1. Open the folder that contains all of your course's files in your editing program.
  2. In the live version of your course, click to view one of the subsections in question.
  3. Note the URL string has a format like this: https://class.stanford.edu/courses/StanfordOnline/OpenEdX/Demo/courseware/Section_one/16f7b57e80e445f9b77a5b9b0f4192b3/

    The slug that precedes the string of letters and numbers is what will correspond to the chapter file name in your course files, so for the above URL, what I will look for in my course files is a chapter file called Section_one.  Find this file in the chapter folder and edit it.
  4. Wrap the two sequentials (which correspond to your subsections) in a <split_test> tag.  The tag you create will look something like this, where you'll specify a user_partition_id value based on what you created in your policy.json file, and you'll specify which group will see which content by providing the location ID for that content.  
    <split_test display_name="A/B Testing between Subsections" user_partition_id="0" group_id_to_child='{"0": "i4x://StanfordOnline/OpenEdX/sequential/621094f6b5294f7aa350511488bdf797", "2": "i4x://StanfordOnline/OpenEdX/sequential/5ee8f79a47674efb93841b88a4942b38"}'>
    Close the <split_test> tag after your two subsections / sequentials.
    </split_test>
