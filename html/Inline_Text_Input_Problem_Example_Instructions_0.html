Inline Text Input Problem Example Instructions

To have an inline text input problem, you can use the following code and edit the values to make it your own.

The thing to note here, though, is that in order for participants to get the problem right with the current code, they will need to get both input fields right before they see a green checkmark in either input, so even if one response they give is right but the other is not, both will show an "X" if they click to submit their answers.

That said, with an additional definition of "acceptable" responses, you can add a little flexibility to what response options are allowed as "correct".

The code below is what you could use to create this kind of problem.  You'll want to edit the content highlighted in red with the answer responses you want to set up as correct. The content in blue is where you can add your own custom message for people to see after an answer is not acceptable, but make sure it does not contain any special characters like quotation marks.  The green-colored content below is what you'll want to edit for your question prompt.

<problem> 
<script type="loncapa/python"> 
def check_function(expect,answer): 
 expect = ['answer1','answer2'] #correct 'expected' answer 
 acceptable = ['answer3','answer4'] 
 ok = False #student is wrong by default 
 overall_message = '' 

#check that inputs are of acceptable form 
 if answer[0] not in acceptable or answer[1] not in acceptable: 
  overall_message = "You must enter answer1 and answer2, or answer3 and answer4, respectively." 
 if answer == expect: 
  ok = True 
  input_list = [{'ok':ok},{'ok':ok}] # one dictionary for each inputfield 
  return {'overall_message':overall_message,'input_list':input_list} 
</script> 

<customresponse cfn="check_function"> 
<p style="display:inline">Enter your question prompt here for the first text input field</p> 
<textline size="12" inline="1"/> 
<p style="display:inline">and more of your question prompt can go here before the second text input field</p> 
<textline size="12" inline="1"/> 
<p style="display:inline">with this bit of text appearing after your secon text input field.</p> 
</customresponse> 

</problem>
