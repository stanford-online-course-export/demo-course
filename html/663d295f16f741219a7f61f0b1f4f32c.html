If you are creating an A/B test between units in a single subsection, you'll want to find the sequential file that corresponds to the subsection.  This sequential will contain two vertical url names that will correspond to the units in that subsection.  Please click here to view detailed instructions for creating an A/B test between units in a subsection.

Creating an A/B test between units within a subsection:

  1. Open the folder that contains all of your course's files in an editing program of your choosing.
  2. In the live version of your course, go to your subsection that contains the two units.
  3. Note the URL string at the end of the URL.  This string will correspond to the sequential file ID in your course's files.
  4. Note the vertical url_name strings in your sequential file.  These will correspond to the units in the subsection.  Here is how that will look before you make any changes:



    Wrap your <split_test> tag around these two (or more) verticals. Your changed file will look something like this:

  5. You can also optionally add a display_name attribute to <split_test>.  The final code would look something like this: 
    <split_test display_name="My A/B Test between Units" user_partition_id="0" group_id_to_child='{"0": "i4x://StanfordOnline/OpenEdX/vertical/db92c4b12dcf43e6a71bc8343a376b7c", "2": "i4x://StanfordOnline/OpenEdX/vertical/1b74b4fe98584d0e8ef55aea917c97eb"}'>
  <vertical url_name="db92c4b12dcf43e6a71bc8343a376b7c"/>
  <vertical url_name="1b74b4fe98584d0e8ef55aea917c97eb"/>
</split_test>
    (Not adding a display_name attribute will result in the unit that now contains the two units in the A/B test displaying a name of random numbers and letters.  This is not something your participants will see.)
  6. Save your changes, then zip up your course file and import this file back into your course by going to Tools > Import.
  7. When you are done, your units in Studio will now exist in a newly created split_test unit, and will longer be editable.  They will look something like this:

  8. The end result in Courseware will look something like this:

  9. Note that participants or students will not see the drop-down menu. Rather, students who end up (note this assignment is randomly assigned and cannot be specified) in one group will see the unit pecified for that group, and students in the other group will only see the unit that has been specified for that group.
  10. If you export and edit your course file again at a later time, you might notice that the sequential file you edited and imported now looks different.
